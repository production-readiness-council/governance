# Public Governance Artifacts

## Purpose

This document defines which governance artifacts must be public and versioned by default.

## Public and versioned by default

The following artifacts should be maintained publicly unless a confidentiality or security exception clearly applies:

- governance model documents
- methodology and workflow rules
- report lifecycle gate definitions
- sponsor non-interference rules
- contributor role criteria and promotion process rules
- governance amendment records
- publication decision artifacts for verdict-bearing reports
- proposal triage rules and public proposal records
- review templates and publication checklists

## Exception rule

If an artifact cannot be fully public, the workflow should still record that a restricted artifact exists, the type of exception, and why full publication is not appropriate.

## Versioning rule

Changes to these artifacts should be made through pull requests so the rationale and review path remain visible.
