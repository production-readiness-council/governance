# Approval and Decision Artifacts

## Purpose

Certain workflow states require a visible artifact rather than an implied status.

## Required public artifacts

The following should have a public GitHub artifact unless confidentiality or security constraints require an exception:

- governance amendments
- proposal triage decisions
- publication decisions for verdict-bearing reports
- role promotion decisions
- major workflow changes affecting review or publication

## Minimum content

An approval or decision artifact should record:

- subject of the decision
- decision maker or deciding body
- date
- rationale
- linked issue, pull request, or governance artifact
- follow-up actions, if any

## Exception handling

If a decision cannot be fully public, the workflow should still record that a restricted-path decision occurred and why the full details are not public.
